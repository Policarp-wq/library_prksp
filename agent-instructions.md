Задача: довести репозиторий https://github.com/Policarp-wq/library_prksp до требований курсовой работы «Приложение для управления библиотекой».

 Ниже — что нужно дописать или переделать. Облачное развёртывание разработчик делает сам, в этот скоуп оно не входит.


1. Каталог книг — администрирование только админом

Сейчас POST /api/books, PUT /api/books/:id, DELETE /api/books/:id доступны любому авторизованному пользователю с owner-based проверкой на изменение. Это противоречит требованию «администратор ведёт каталог».

Изменить: на эти три эндпоинта добавить middleware-проверку req.user.role === "admin", иначе возвращать 403. Owner-based логику убрать. Поле owner_id оставить как метаданные о том, кто внёс книгу, либо удалить миграцией — но в API оно больше не должно влиять на доступ.


2. Поиск по каталогу

GET /api/books сейчас возвращает все книги без параметров. Добавить query-параметры:
- q — общий поиск по подстроке в title и author, case-insensitive;
- title и author — отдельные фильтры (опционально).

Реализовать через ILIKE '%' || $1 || '%' в SQL. Без q — полный список как сейчас.

Валидация query: максимальная длина 100 символов, разрешить буквы, цифры, пробелы, "-", "'", ".". Падающие запросы — 400.


3. Сущность loans (выдача и возврат книг)

Создать таблицу loans:

CREATE TABLE IF NOT EXISTS loans (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id),
  book_id INTEGER NOT NULL REFERENCES books(id),
  issued_at TIMESTAMP NOT NULL DEFAULT NOW(),
  returned_at TIMESTAMP NULL
);
CREATE INDEX idx_loans_book_active ON loans(book_id) WHERE returned_at IS NULL;

Эндпоинты:

POST /api/loans (auth user) — body { bookId }. Проверить, что книга существует и нет активного loan на неё (returned_at IS NULL). Если занята — 409. Если свободна — создать loan от имени req.user.id, вернуть 201 + объект.

PATCH /api/loans/:id/return (auth user) — отметить возврат. Разрешить только владельцу loan (user_id === req.user.id) или админу. Если loan не существует — 404, если уже возвращён — 409.

GET /api/loans (auth user) — для admin вернуть все loans, для user — только свои. Поддержать query ?active=true для фильтрации только текущих (без даты возврата).

В GET /api/books добавить в каждую запись поле available: boolean — есть ли активный loan на эту книгу.


4. Сидинг

В стартовой инициализации БД добавить 1-2 тестовых loan от пользователя user на одну из стартовых книг (одна возвращена, одна активная). Это нужно для демонстрации в скриншотах курсовой.


5. Валидация

В node-postgres/server.js добавить validateLoanPayload({ bookId }) — bookId должен быть positive integer. Использовать parsePositiveIntId для :id в loans-эндпоинтах.


6. Фаззинг-тесты

В node-postgres/__tests__/validation.fuzz.test.js добавить describe-блок для валидации loan payload: подавать рандомные значения и проверять, что валидатор отбивает всё кроме positive integer. Использовать существующий стиль через fc.assert и fc.property.


7. Frontend

Управление каталогом (кнопки «Добавить книгу», «Изменить», «Удалить») показывать только если у пользователя role === "admin". Для обычных пользователей оставить только просмотр и кнопки «Взять» / «Вернуть».

В странице каталога добавить поле поиска, отправлять ?q=... на бэкенд.

В странице профиля или отдельной странице «Мои книги» — список активных loans пользователя с кнопкой возврата.
